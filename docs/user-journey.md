# User Journey

![User Journey](./images/user-journey.png)

This diagram shows the typical workflow of a doctor using CalmAnchor Lite throughout the day. It focuses on reviewing upcoming appointments, accessing patient information when needed, and safely rescheduling appointments while avoiding time conflicts.

### Journey Flow

The primary workflow begins when the doctor opens the application to review the day's appointments. From the Day Schedule screen, the doctor can select an appointment to view the patient's medical information before the consultation.

If a patient requests a different appointment time, the doctor opens the rescheduling screen. The application retrieves the available time slots for that day, excluding any appointments that have already been booked. After selecting a new time, the appointment is updated in the database and the revised schedule is displayed.

This workflow represents the primary interaction within CalmAnchor Lite. Alternative navigation paths, such as browsing the complete patient list or viewing the doctor's profile, are also available but fall outside the application's core appointment-management journey.
